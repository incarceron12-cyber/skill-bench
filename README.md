# skill-bench

A compounding research system for designing high-signal benchmarks of agentic knowledge work.

The project has three goals:

1. **Understand the state of the art** in long-horizon agent / knowledge-work evaluation.
2. **Convert domain expertise into benchmarkable tasks** that reveal what AI agents can and cannot do.
3. **Compound over time**: daily paper ingestion, structured paper reviews, evolving taxonomies, benchmark design notes, and GitHub history.

This repository is intentionally both a research notebook and an implementation scaffold. It should become more useful every day.

## Core thesis

The next useful AI benchmarks will not look like isolated Q&A. They will look like messy work: fragmented files, contradictory sources, partial context, spreadsheets, decks, memos, product specs, analysis notebooks, and decisions under ambiguity.

`skill-bench` explores how to build those benchmarks on a limited budget.

## Repository map

| Path | Purpose |
|---|---|
| `docs/` | Research agenda, benchmark design, incentive mechanisms, synthesis docs |
| `papers/` | Structured paper reviews and topic indexes |
| `data/papers/` | Machine-readable metadata, downloaded PDFs/text, discovery state |
| `templates/` | Review, task, rubric, and sponsor templates |
| `scripts/` | Paper discovery, download, extraction, indexing, report generation |
| `benchmarks/` | Future pilot benchmark scenarios and specs |
| `reports/daily/` | Daily research digests generated by cron jobs |

## Daily compounding loop

The intended loop is:

1. Search new papers and benchmark releases across arXiv, Semantic Scholar, Artificial Analysis, GitHub, and curated sources.
2. Download and extract papers where possible.
3. Write structured reviews focused on:
   - unique insight,
   - methodology,
   - scoring/evaluation design,
   - relevance to agentic knowledge-work benchmarks,
   - what we can steal/adapt.
4. Update the taxonomy and benchmark-design notes.
5. Commit and push changes to GitHub.
6. Send Samuel multiple concise but substantive Telegram updates per day.

## Design stance

We are not trying to make a cheap clone of AA-Briefcase. We are trying to learn the deep design principles behind credible agentic benchmarks and find niches where a smaller, open, sponsor-supported project can be unusually useful.
