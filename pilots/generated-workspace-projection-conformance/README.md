# Generated workspace projection conformance

This bounded, zero-call pilot falsifies generated workspaces **before** any agent
trial. It spans an evidence-grounded incident memo and a structured laboratory
inventory reconciliation. Both are builder-authored synthetic shapes; neither is
presented as representative professional work.

## General hypothesis and charter fit

The slice advances charter objectives **B** (expertise-to-evaluation boundaries)
and **C** (executable infrastructure). It tests the cross-domain hypothesis that
a rich generated workspace is benchmark-eligible only when independently frozen
profile, plan, file graph, byte tree, objective, authority, and evidence-status
projections survive mechanical and semantic falsification. Useful completion is
exact localization of every predeclared defect plus byte-stable preservation of
unrelated projections after repair. Passing does not establish ecological or
professional validity.

## Frozen prospective design

`suite.json` was written and independently hashed before `audit.py` was
implemented. It contains two clean controls and ten balanced planted defects
(five per work shape): projection hash drift, dangling graph endpoint, stale
derivation, impossible objective, semantic drift, unsupported professional
claim, derivation cycle, valid-time inversion, retrieved-to-synthetic fallback
laundering, and collaborator authority overreach.

The records reuse repository concepts rather than fork schemas:

- task projection identity and independent canonical hashes;
- persistent-workspace paths, dependency relations, valid times, and immutable
  state;
- artifact/evidence-view separation through source and derived byte records;
- authority/evidence-chain status and bounded claims;
- task-health-style fail-closed pre-trial conformance.

A standalone pilot record is used because the existing bundle is one task at a
time and this experiment is a cross-task generator suite. No existing schema
obligation proved unrepresentable, so no schema was extended.

The source review is
`papers/agent-benchmarks/2026-07-13-synthetic-computers-workspace-validity.md`.
The immutable PDF/text and pinned release-provenance hashes are embedded in
`suite.json` and verified by tests.

| Endpoints, cycles, valid time, stale derivations | Repair 3 and falsifiable experiment (lines 232, 244–253) |
| Retrieved versus synthetic fallback status | Method §1 and Repair 4 (lines 63–65, 233) |
| Professional-claim and collaborator authority | Method §2, Repair 1, limitations 3/9 (lines 67–71, 178–184, 230) |

The exact semantic-atom representation and deterministic mutations are explicit
**internal design hypotheses**, not claims made by the paper.

## Replay

```bash
python pilots/generated-workspace-projection-conformance/audit.py
python -m unittest tests.test_generated_workspace_projection -v
```

`report.json` contains machine-readable boundary findings, exact-localization
flags, unrelated-projection preservation, and clean-baseline repair checks. The
replay uses no model, network, or agent execution.

## Claim ceiling

The only licensed claim is the exact deterministic behavior of these twelve
frozen builder-authored cases. Excluded claims include generator realism,
occupational representativeness, expert/professional validity, human work
duration, agent capability, production fitness, and deployment readiness.
