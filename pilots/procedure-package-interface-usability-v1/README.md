# Procedure-package interface usability v1

This is a prospective, task-blind **interface conformance** study, not a fourth procedure-transfer study. It freezes one guide-plus-unrelated-example serialization strategy and three structurally varied synthetic source corpora before any model call. `case-heldout` is designated held out before outcomes.

The general hypothesis is that an explicit representation boundary—especially the requirement that structured source primitives become JSON strings—can make the existing independent package contract usable without downstream task hindsight. The reusable artifact is the frozen model-facing interface, launcher, independent adjudication, field-level report, and mutation-tested anti-edit boundary.

The cases are builder-authored internal calibration only. There are no downstream tasks, private checks, controls, assignments, executor calls, or professional sources in this pilot. The interface example demonstrates JSON shape and typing only and is not authority for any measured case.

Before calls:

```bash
python pilots/procedure-package-interface-usability-v1/preflight.py --check-paths --report pilots/procedure-package-interface-usability-v1/preflight-report.json
python -m unittest tests.test_procedure_package_interface_usability_v1 -v
```

Generation is authorized only after the frozen protocol commit is pushed and the preflight and included-USD-0.00 provider gates pass. Exactly one source-only attempt is allowed per case, with no repair or retry. Every raw prompt, output, usage record, stdout, stderr, and independent adjudication is retained. Strict intended, attempted, service-valid, and schema-valid denominators are reported separately.

All seven claim ceilings remain false: expert provenance, professional validity, transfer, agent capability, utility, production fitness, and readiness.
