# Realistic-artifact action-boundary composition v1

**Status:** prospectively frozen internal synthetic validation package. It composes two existing, materially different artifact tasks with one directly observed synthetic state transition each. It does not establish retained-state causality, general capability, expert/professional validity, safety, privacy, production fitness, readiness, or cross-domain generalization.

## Charter decision filter

- **Objectives:** B (expertise-to-evaluation methodology) and C (executable infrastructure).
- **Artifact/evidence:** six-cell protocol, zero-call isolation and mutation preflight, one-shot sandboxed executions, native artifacts, state deltas, usage, and exact replay.
- **Uncertainty:** whether current versus stale/revoked/conflicting authority can govern the first consequential effect while substantial evidence analysis and artifact work still proceed.
- **Mode:** building and validation.
- **Duplication/scope:** reuses action semantics from workspace v4 and two existing pilots; it is not a vendor-response or benchmark-adoption scope commitment.
- **Useful completion:** all six intended cells execute once after a pushed freeze and produce interpretable native-artifact/action evidence, or fail closed in the strict denominator.

## Design

The matrix crosses two work shapes (`vendor`, `lh`) with current-authorized, stale-or-revoked, and conflicting authority. Within each work shape, the public task and source pack are byte-identical across cells; only the declared `authority.json` differs. Every cell requires both native artifacts before an `action-decision.json`. The runner realizes an endpoint transition only for an exact, schema-readable effect request and independently records before/after state.

The vendor arm retains the existing incident workspace and memo/JSON plan. The LH arm retains the evidence CSV/manifest, spreadsheet-compatible evidence matrix, and recommendation memo. Native artifact checks are separate from authority use, exact execution, pre-effect restraint, and recovery. Thus neither schema-only nor artifact-only completion passes.

`protocol.json` pins parent Git trees, every source-pack byte, builder/grader/runner/launcher hashes, configured system, budget, order, no-retry rule, public basis, valid time, targets/parameters, prohibited effects, safe partial progress, recovery categories, observer, denominator, and claim ceiling. Parent directories are never edited.

## Run order

```bash
python pilots/action-boundary-composition/v1/run.py verify
python pilots/action-boundary-composition/v1/run.py preflight --require-pushed
python pilots/action-boundary-composition/v1/run.py execute
python pilots/action-boundary-composition/v1/run.py replay
python -m unittest tests.test_action_boundary_composition_v1 -v
```

The preflight makes zero model calls and checks task-scoped cwd, allowlisted visibility, private/repository denial, output-only writes, source/implementation hashes, unchanged parent trees, strict denominators, coherent controls, and planted hidden-label/oracle, public/private mismatch, authority laundering, wrong narrative/state, late effect, wrong target/parameters, destructive partial work, schema-only success, artifact-only success, observer insufficiency, and denominator drift.
