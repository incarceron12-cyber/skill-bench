# Delayed-obligation dual-task pilot

This **prospective internal synthetic pilot** repairs the PM-Bench construct gap by grading a primary knowledge-work artifact and a delayed obligation separately. It reuses two purposive work shapes: a document/policy disposition and a quantitative release decision. Each equivalent form has one publicly authorized delayed obligation, a later replacement/update, an obsolete-action lure, a visible due cue, and primary-artifact preservation checks.

Three evidence treatments are crossed under the same three-turn, file-only, isolated configured system and budget:

- `neutral_interrupt`: a non-answer-bearing interruption; the obligation channel is visible only if the agent queried it on the preceding turn.
- `channel_hint`: names the obligation channel but not its contents; access still requires a preceding query.
- `oracle_reminder`: evaluator-derived reminder exposing the current update or due action. This is privileged evidence, not an internal-memory aid.

The deterministic grader keeps encoding/retention, cue monitoring/access, update reconciliation, due-status inference, timing, action realization, obsolete/lure commission, primary-artifact quality, and collateral loss separate. `instrument/calibration-cases.json` contains perfect play and planted failures for every signature. All obligations and updates originate in agent-visible public events; private expected state reaches the agent only through the predeclared oracle treatment.

Commands:

```bash
python pilots/delayed-obligation-dual-task/freeze_protocol.py
python pilots/delayed-obligation-dual-task/run_study.py preflight
# freeze commit must be on origin/main before calls
python pilots/delayed-obligation-dual-task/run_study.py execute
python pilots/delayed-obligation-dual-task/run_study.py replay
```

This instrument can support exact conformance and bounded attempt observations only. It cannot establish general prospective memory, a causal reminder effect, professional capability/validity, safety, production fitness, or readiness. Shapes are never pooled.

## Provenance and rationale

Fields and failure stages instantiate the obligation state machine and repair recommendations in `papers/agent-benchmarks/2026-07-16-pmbench-prospective-memory-validity.md` §§ “Unique insight,” “Transfer,” and “Concrete repository actions” (review lines 80–97 and 144–179). Primary-artifact checks and frozen configured-system identity reuse `pilots/evidence-request-receipt-repair-v3/` and the execution-validity boundary in `docs/benchmark-design-taxonomy.md`. The two synthetic shapes test reusable machinery; they are not domain scope declarations or expert-authored professional cases.
