# Pre-task procedure transfer v1: fail-closed feasibility slice

This prospective internal-calibration slice asks whether one procedure generated
before a task is revealed can be reused on untouched forms under an equal
execution envelope. It advances charter objectives B/C by testing the
source→package→adoption→artifact chain across two unlike synthetic task
families; neither family defines benchmark scope.

The zero-call audit found a decisive precondition failure. The four candidate
forms are untouched by model execution, but each public task already discloses
the full authority/scope/supersession/threshold/consequence procedure. There is
no separately versioned family-level source-authority corpus from which to
generate a package. Consequently, the nominal no-package/no-raw-context arm
would still receive the complete procedure, while a newly authored “raw
context” would either duplicate the task or change the instrument after seeing
its forms. That cannot estimate pre-task source distillation or reuse.

`protocol.json` freezes the intended eight-arm contrast, equal-envelope policy,
firewalls, accounting, adoption observations, and claim ceilings.
`feasibility-report.json` preserves the exact pre-call blocker and all skipped
gates. `validate.py` verifies hashes and fail-closed consistency; tests mutate
the treatment set, claim ceiling, eligibility, and leakage scrub.

No package, task, or testimony was fabricated to force feasibility. No model or
provider call occurred.

```bash
python pilots/pretask-procedure-transfer-v1/validate.py --check-paths
python -m unittest tests.test_pretask_procedure_transfer_v1 -v
```
