# Analytical hypothesis lifecycle conformance v1

This is a **zero-call, builder-authored validation package**, not a benchmark result or a reconstruction of PM-LLM. It advances charter objectives B and C by testing a reusable inquiry graph across two unlike work shapes: a vendor-incident decision memo and a laboratory-quality quarantine handoff. Neither domain defines `skill-bench`'s scope.

## General hypothesis

A fluent causal narrative must not receive verified-analysis credit unless the record preserves:

`source-bound observation → plausible candidate + rival → predeclared discriminating test → executed result → contradictory-evidence adoption → update/rejection → bounded conclusion → authorized consequence or escalation`.

`protocol.json` freezes both evidence graphs, one plausible unsupported cause per case, rivals, contradictory evidence, discriminating and non-discriminating executable tests, update rules, authorization boundaries, configured-system identity, two repeats, and claim limits before execution. `observations.json` retains every raw synthetic analyst output and every raw stage-observer output. The fourth attempt contains a deliberately malformed conclusion-observer output; it is typed `invalid_output` and excluded only from the conclusion denominator rather than converted to failure.

## Executable artifact

```bash
python pilots/analytical-hypothesis-lifecycle-v1/replay.py --check-paths --write-report
python -m unittest tests.test_analytical_hypothesis_lifecycle -v
```

The replay independently executes rate-gap and stratified-rate-gap tests from frozen rows. It reports **five non-interchangeable score families**—candidate quality, test validity, evidence adoption, final conclusion, and consequence—with criterion evidence and separate intended/valid/invalid denominators. It emits no holistic scalar and does not pool the two domains.

The retained matrix demonstrates:

- a sound candidate/rival/test/update/escalation chain in each domain;
- a plausible cause promoted to truth before a valid test;
- selection of a non-discriminating test;
- failure to adopt contradictory evidence;
- an unsupported harmful action despite a correct descriptive observation;
- a bounded escalation path; and
- typed evaluator invalidity without substantive zero imputation.

## Provenance and contract reuse

The design basis is the complete PM-LLM paper/release review at `papers/agent-benchmarks/2026-07-17-pm-llm-process-analysis-judge-validity.md`, especially its analytical inquiry ladder and concrete cross-domain changes. The underlying release audit is pinned in `protocol.json`. The synthetic domain records, hypotheses, thresholds, and actions are explicit builder-authored design hypotheses—not paper data, expert testimony, or professional policy.

No new schema was added. Hash-pinned contracts are reused for evidence identity/adoption/stopping/consequence, configured-system and trace vocabulary, metric denominator/invalidity semantics, and bounded validity claims.

## Claim boundary

The report licenses only deterministic detection of the planted lifecycle successes, failures, and evaluator invalidity in these frozen records. It does **not** establish agent capability, causal correctness outside the fixture, cross-domain generalization, process-mining capability, expert or professional validity, an intervention effect, production fitness, or deployment readiness.
