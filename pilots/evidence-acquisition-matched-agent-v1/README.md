# Matched evidence-acquisition agent slice v1

This bounded validation executes one frozen configured agent across two unlike internally authored synthetic work shapes and three matched information conditions, with two declared repeats per shape-condition cell. It tests whether the evidence-acquisition episode contract can retain real request intent, deterministic parser/access transitions, evidence uptake, stopping, artifacts, usage, and separate endpoint checks.

`expert_minimal` is a condition label only. Its set is builder-authored and synthetic; no expert selected, reviewed, or approved it. The two purposive shapes are reported separately and are not evidence of cross-domain generality.

## Prospective controls

`protocol.json` freezes scenario graphs, prompt, parser/access policy, grader, runner, schedule/order seeds, configured system, budgets, invalidity rules, stopping/loss rules, checks, and claim ceiling before outcomes. The frozen commit must exist on `origin/main` before `run_study.py execute` permits model calls. Every scheduled attempt has one retained trajectory and no retry/replacement.

Preflight runs zero-call filesystem isolation and feedback-firewall canaries. Active turns expose only public context, prior request/access messages, released content, and budget state. Full-information supplies all admissible/delayed atoms at start; synthetic-minimal supplies exactly marked atoms. Private scenario answers, grader, protocol, parser implementation, other attempts, and repository are inaccessible.
