# Workspace evidence-closure matched treatment v1

A prospectively frozen, internal configured-agent study of whether a durable obligation/check workspace changes independently graded outcomes. It uses two unlike synthetic work shapes: native JSON record repair and prose handoff repair.

The intervention exposes only `instrument/public-guide.md`; task/source inputs, configured model/provider/version, file-only tools, one-call budget, timeout, stopping rule, grader, and invalid policy are fixed within each pair. Private checks and expected values remain outside the sandbox. Two nonce-labeled repeats per shape/condition are retained without replacement; the provider does not expose seed control.

Outcomes remain separate: artifact completion, independent correctness, obligation coverage, predicate satisfaction, source entailment, execution-to-byte lineage, report freshness, repair retention, newly introduced errors, latency, tokens/cost observability, invalids, and within-shape paired differences. This purposive synthetic study cannot establish expertise transfer, universal workflow efficacy, professional validity, reliability, production fitness, or readiness.

Run `python pilots/workspace-evidence-closure-treatment-v1/run_study.py preflight`, then (only after `protocol.json` is frozen on `origin/main`) `... execute`, and `... replay`.
