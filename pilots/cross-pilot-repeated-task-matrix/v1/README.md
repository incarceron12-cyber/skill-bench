# Repeated cross-pilot task-family matrix v1

This prospective internal calibration package tests whether one exact configured system can be repeated on multiple forms within two unlike knowledge-work families: evidence reconciliation and persistent incident-state handling. It advances charter objectives B/C by making within-form repeat stability distinguishable from between-form and between-family transport.

All scenarios, thresholds, organizations, and expected decisions are builder-authored hypotheses. They are not expert testimony or evidence of professional practice. The reusable object is the frozen form/repeat/criterion/severity matrix, not either domain.

`protocol.json` is the authority for order, seeds, eligibility, service-failure handling, confidence-channel handling, and claim ceilings. Private `answer-key.json` files are grader-only. No execution may begin until the exact frozen package is committed and pushed, all five zero-call/pre-call gates pass, and provider cost remains included at USD 0.00.
