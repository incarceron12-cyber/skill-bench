# Grading, Validity, and Benchmark-Lifecycle Evolution

**Purpose.** This matrix compares scoring families as repairs to different measurement failures. It advances charter objectives A–D by identifying what `skill-bench` should retain, repair, and directly test. It is not a claim that newer or more complex graders are intrinsically better. Evidence comes from the linked full-source reviews and inspected releases; design intent and demonstrated reliability remain separate.

## Cross-family conclusions

### 1. No grader family dominates; each observes a different predicate

Exact matching and executable checks are strongest where a complete computable consequence exists. State checks admit alternative paths but observe only selected environment predicates. Rubrics, model judges, and experts can assess open-ended professional quality but introduce evidence-view, authority, framework, and rater effects. Psychometric models describe response/rater structure but do not confer construct validity. Validity arguments license claims but do not produce measurements.

**Decision:** `skill-bench` should preserve a typed observation graph rather than search for one universal scorer:

`criterion → required evidence view → observer/configuration → raw observation → validity/adjudication status → metric → licensed claim`.

### 2. “Objective” means inspectable predicate, not objective construct

LiveBench's keys and code tests eliminate many judge biases, but substring acceptance, canonical order, detectable instruction constraints, selected tests, and symbolic parsers still encode design choices. SaaS checkpoints and browser states are equally authored. A deterministic function can be perfectly repeatable and still omit the professional consequence of interest.

**Decision:** retain deterministic checks as high-value witnesses, pair them with verifier challenge sets and bounded claims, and preserve expert/artifact evidence for criteria that cannot be reduced legitimately.

### 3. Grader repair and benchmark renewal both change the instrument

Parser expansion can correctly repair false negatives, but historical scores then have two legitimate versions. LiveBench's current AMPS fallback also shows a grader can cross families—from symbolic equivalence to a model judge—inside one evolving repository. Replacing easy tasks changes the administered population even if model ranks remain stable.

**Decision:** every form, prompt, grader, evidence view, aggregation, and configured system receives an immutable identity. Regrading creates linked observations; renewal requires anchors or equivalent-form evidence before absolute trend claims.

### 4. Agreement is not validity, and stability is not equivalence

Judge-human agreement can conceal shared cues, severity, insufficient evidence, or framework differences. Rank stability across forms can coexist with deliberate difficulty drift. A response matrix or Rasch fit can improve comparability only relative to its linked panel and assumptions.

**Decision:** report agreement, severity/fit, repeat stability, equivalent-form linkage, construct evidence, decision loss, cost, and audit burden separately.

### 5. Benchmark lifecycle is part of measurement

Live forms expose four clocks: source, exposure, instrument, and configured system. Task-health systems add role transitions, saturation, defect adjudication, revision, and retirement. Contamination is therefore not a binary item flag; it is an information-flow and role history.

**Decision:** task records must retain creation/source time, visibility and access history, candidate/admission history, public/private/calibration/regression role, model/tool exposure, grader revisions, bridge items, retirement reason, and claims permitted at each stage.

## Minimum comparative validation program for skill-bench

1. **Deterministic grader conformance:** positive, negative, malformed, alternate-valid, substring/reward-hacking, missing-view, and environment-invalid cases.
2. **Observer-view study:** artifact-only vs trace vs environment query vs source pack for the same criterion.
3. **Plural-grader calibration:** deterministic, model, and expert observations retained independently; estimate predicate-specific confusion, rater effects, repeats, and audit cost.
4. **Form/grade bridge:** fixed configured systems run repeatedly on old/new forms and old/new graders with frozen anchors; report rank and absolute shifts separately.
5. **Exposure intervention:** matched fresh/private, intentionally exposed, and post-release forms under fixed retrieval policy; record exposure, visibility, adoption, and effect.
6. **Claim challenge:** validity validator must reject professional-capability/readiness claims when evidence supports only a selected predicate, relative preference, within-form ranking, or panel-relative severity.

## Scope and evidence boundary

This synthesis compares reviewed methods; it does not claim every family has equal empirical maturity. LiveBench, SaaS-Bench, ResearchRubrics, AgentRewardBench, and several professional/interactive benchmarks include inspectable releases. ECBD and Validity-Centered Evaluation primarily contribute design frameworks. Rater-effect evidence is narrow and panel-relative. The Anthropic lifecycle source is production guidance without public quantitative validation. These differences are why each row separates implemented repair from demonstrated reliability.
